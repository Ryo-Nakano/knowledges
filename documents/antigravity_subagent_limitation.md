# Antigravityにおけるサブエージェント作成・利用の制限について

**結論:**
現状の統合プラットフォームとしての「Antigravity」においては、ユーザーが独自の「サブエージェント（ネイティブで独立した自律的エージェントインスタンス）」を新規作成・利用することは**仕様上不可能**です。

## 詳細および仕様上の制約

1. **Mission Controlによる自動管理**
   Antigravity本体は、「Browser Subagent」や「Terminal Subagent」など、システムに標準で組み込まれているサブエージェントを自動的に呼び出すアーキテクチャ（Mission Control）を採用しています。ユーザーが任意の機能を持つ新規のサブエージェントを追加構築し、同等に稼働させる機能は提供されていません。

2. **Gemini CLIとの混同に注意**
   基盤技術である「Gemini CLI」には、`.gemini/agents/*.md` で定義できる実験的なサブエージェント機能が存在しますが、これはGemini CLIのコマンドライン用途に限定されたものであり、Antigravityという枠組み全体でシステムネイティブなサブエージェントとして統合できるわけではありません。

## 代替となるアプローチ（拡張手法）
Antigravity環境において、AIエージェントにカスタム機能や専門的なタスクを委譲させたい場合は、サブエージェントを作成するのではなく、以下の手段をとるのが公式かつ一般的なアプローチとなります。

- **Skill（スキル）の作成**
  `.agent/skills/<skill-name>/SKILL.md` を作成し、AI（メインエージェント）に特定の状況における手順やルールを指示しておく方法。
- **Workflow（ワークフロー）の定義**
  `.agent/workflows/` 以下に手順書を定義し、タスクの順序や実行内容を自動化する方法。
- **MCP Serverの連携**
  Model Context Protocol (MCP) を満たす外部のサーバープログラムを用意し、設定を連携させて外部ツールとして呼び出す方法。
