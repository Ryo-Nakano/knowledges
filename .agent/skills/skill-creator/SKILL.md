# Skill Creator

このスキルは、Antigravity の新しいカスタムスキルを対話形式で作成するプロセスをガイドする。  
ユーザーが各項目の答えを持っているとは限らないため、**選択肢と推奨理由を提示しながら**一緒に決めていく。

> **重要：ユーザーとのすべてのやり取りは日本語で行うこと。**

---

## 前提知識の読み込み

このスキルを開始したら、まず `references/skill_template.md` と `references/examples.md` を読み込む。  
これらはスキル生成の雛形と完成例を含んでおり、高品質な SKILL.md を作るために必須。

---

## ワークフロー

### Phase 1: キックオフ

ユーザーに以下を伝えてヒアリングを始める：

```
スキル作成を始めましょう！いくつか一緒に決めていきたいことがあります。
まずは「どんなことをしてほしいスキルですか？」を教えてください。
自由に話しかけてもらえれば、あとは一緒に整理していきます。
```

---

### Phase 2: ヒアリング（提案型）

以下の各項目を **1〜2項目ずつ** 確認する。  
ユーザーが答えを持っていない場合でも前に進めるよう、**推奨する選択肢とその理由を添えて提案する**。  
「こういう目的・意図のスキルなら、〜がよいと思います。理由は〜です。いいですか？」という形で進める。

#### 2-1. スキルの目的・意図（必須）

ユーザーの言葉から以下を引き出す：

- このスキルは**何をするか**（例: コードをレビューする・仕様書を書く・環境セットアップを自動化する）
- **どんなときに使うか**（ユーザーがどんな言葉を発したら発動してほしいか）

> ここはユーザーに答えてもらう必要がある唯一の必須項目。それ以外は提案で補える。

#### 2-2. ワークフローの粒度

ユーザーの目的をもとに、以下を提案する：

| パターン | 内容 | 向いているケース |
|---|---|---|
| **シンプル型** | 指示を1〜3ステップで完結させる | 繰り返し使う単純な処理（フォーマット変換・翻訳など） |
| **ガイド型** | 複数フェーズで段階的に進める | 意思決定が必要なタスク（設計・レビュー・作成など） |
| **インタラクティブ型** | ガイド型 + ユーザーへの確認を随所で挟む | 安全性が重要・判断がユーザー依存のタスク |

「〜の目的であれば**ガイド型**が向いていると思います。なぜなら〜だからです。この方向でいいですか？」と提案する。

#### 2-3. 配置場所

ユーザーの目的・対象範囲をもとに以下を提案する：

| 選択肢 | パス | 推奨するケース |
|---|---|---|
| **プロジェクト固有** | `.agent/skills/<name>/` | このリポジトリ・プロジェクト専用のスキル |
| **グローバル** | `~/.gemini/antigravity/skills/<name>/` | どのプロジェクトでも使いたい汎用スキル |

例：「このスキルは特定のプロジェクトに依存しない汎用的な用途に見えます。**グローバル配置**にすることで、他のプロジェクトでも使えるようになります。グローバルにしますか？それともプロジェクト固有にしますか？」

#### 2-4. スキル名

以下のルールに従い、ユーザーの目的から**候補を複数提案**する：

- 小文字・ハイフン区切りのみ（例: `code-reviewer`, `doc-generator`）
- 目的を端的に表す名詞+動詞構成が読みやすい
- 既存スキル名との重複を避ける（`read_file` や `list_directory` で確認する）

例：「目的から考えると `spec-writer` または `requirements-helper` はいかがでしょう？」

---

### Phase 3: SKILL.md ドラフトの生成

ヒアリング内容と `references/skill_template.md` をもとに SKILL.md のドラフトを作成し、ユーザーに提示する。

**生成時の注意点：**

- `description` は三人称・具体的なキーワードを含める。Antigravity がルーティング判断に使う最重要フィールド。
  - 悪い例: 「スキルを作るときに使う」
  - 良い例: 「Antigravity のカスタムスキル（SKILL.md）の作成を支援する。ユーザーが〜と言ったときに使用する。」
- システムプロンプト（本文）には、役割・ワークフロー・制約を明確に記述する
- `references/examples.md` の完成例を参考に、具体的で実用的な内容にする

ドラフト提示後：
```
このような内容でいかがでしょう？
変えたい部分や追加したい内容があれば教えてください。
問題なければファイルを作成します。
```

---

### Phase 4: ファイルの作成

ユーザーの承認を得てから `write_file` でファイルを作成する。

**作成するパス：**
- プロジェクト固有: `<workspace-root>/.agent/skills/<name>/SKILL.md`
- グローバル: `~/.gemini/antigravity/skills/<name>/SKILL.md`

**作成前のチェック：**
1. 同名のスキルが既に存在しないか `list_directory` で確認する
2. 存在する場合は上書きの可否をユーザーに確認してから実行する

作成後：
```
✅ スキルを作成しました: <ファイルパス>

このスキルを使うには：
- Antigravity を再起動するか、新しいセッションを開始してください
- 「<スキルの description にあるキーワード>」のような言葉で呼び出せます
```

---

## 改善ループ

このスキルの使用中に問題が発生したら、原因と対策を `references/known_issues.md` に記録する。  
次回の起動時にそのファイルを読み込んで過去の知見を活かす。
