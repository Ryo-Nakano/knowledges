# Readable Code Checker

このスキルは、指定されたコードが『リーダブルコード（The Art of Readable Code）』の完全観点まとめ（`readable_code_complete_guide.md`）に準拠しているかをレビューするための専門的なワークフローを提供します。

## 対象読者
コードの可読性を高めたい開発者、および可読性チェックを委譲されたメインエージェントやサブエージェント。

## ワークフロー

**重要:** ユーザーとのやり取りは必ず**日本語**で行ってください。

1. **対象の特定**: ユーザーに対して、レビュー対象のファイルやディレクトリ、あるいはコードスニペットを指定してもらいます。
2. **ガイドラインの参照**: 同梱されている `references/guide.md`（リーダブルコードの完全観点まとめ）を `read_file` ツールで読み込み、最新の原則（命名、書式、コメント、制御フロー、関数の分割、アーキテクチャへの昇華など）をインプットします（もし既に記憶している場合はスキップ可）。
3. **コードの調査と評価**:
   - `read_file` や `grep_search` 等のツールを用いて、対象のコードを精査します。
   - `guide.md` の各観点（Part 1: 表面上の改善 〜 Part 5: 全体設計）と照らし合わせ、違反している箇所や改善余地のある箇所を特定します。
4. **レビュー結果の提示**:
   発見した改善点ごとに、以下のフォーマットでわかりやすく報告します：
   - **箇所**: どのファイルの何行目か、あるいはどの関数/変数か。
   - **指摘理由**: なぜガイドラインに反しているのか（例：「汎用的な名前 `tmp` が使われており、意図が伝わらないため」）。
   - **改善案（Before/After）**: 具体的にどう修正すべきかをコードブロックで提示します。
5. **リファクタリングの提案**:
   レビュー報告の最後に、「提案したリファクタリングを私が（`replace`ツール等を使って）コードベースに直接適用しましょうか？」とユーザーに尋ね、許可を得てから実際のファイル変更を行います。
