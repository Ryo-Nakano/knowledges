# Auto Japanese Committer

このスキルは、Gitリポジトリの変更内容を分析し、適切な日本語のコミットメッセージを作成した上で、コミットとプッシュまでを自動化するためのガイダンスを提供します。
また、変更内容に応じて「コミットを分けるべきか」を判断し、ユーザーに提案する機能も備えています。

## ワークフロー

### 1. ガイドラインの確認
- まず、本スキルに同梱されている `references/git-commit-guidelines.md` を読み込みます。このガイドラインがコミット粒度とメッセージの判断基準となります。

### 2. 変更内容の確認
- 以下のコマンドを実行して、現在の変更内容（ステージング済み、未ステージング）と最近のコミットログを確認します。
  ```bash
  git status && echo "\n--- DIFF ---" && git diff HEAD && echo "\n--- LOG ---" && git log -n 3
  ```

### 3. コミット分割の判断と自動実行
- `references/git-commit-guidelines.md` の基準（関心の分離、アトミックコミットなど）に照らし合わせ、現在の変更内容を1つのコミットにまとめるべきか、複数に分割すべきかを判断します。
- **分割すべきと判断した場合:**
  - ユーザーの許可や回答を待たず、AIは自動的に論理的な単位（ファイル単位、または必要に応じてファイルの部分的な変更単位）で分割して処理を進めます。
  - 実行する前に、「[具体的な理由]のため、[Aの変更]と[Bの変更]のN個のコミットに分割して処理を実行します」とユーザーに報告（宣言）だけを行ってください。
  - AIは自動的に `git add` と `git commit` を複数回繰り返し実行します。Hunk単位での分割が必要な場合は、`git diff` の内容をもとにパッチを作成して `git apply --cached` でステージングするか、一時的に作業ファイルを書き換えて論理的な単位でコミットを繰り返すなどの方法を用いて、分割コミットを実行してください。すべての分割コミットが完了したら、最後にまとめて `git push` を実行します。
- **分割不要と判断した場合:**
  - 事前の報告は行わず、すべての変更を1つのコミットとしてそのままコミット・プッシュを実行します。

### 4. コミットメッセージの生成
- 変更内容（diff）を元に、日本語でコミットメッセージを考えます。
- 以下のルールに従って作成してください：
  - Conventional Commits のプレフィックス（feat, fix, chore, docs, refactor, test, style など）を使用する。
  - 1行目は簡潔な要約（50文字以内）。
  - 2行目以降に詳細な変更理由や内容を箇条書きで記載する。
  - 過去のコミットログ（`git log`）のスタイルに合わせる。

### 5. コミットとプッシュの実行
- 分割不要と判断したケースでは、考案したメッセージを使用して一括でコミットとプッシュを実行します。
  ```bash
  git add .
  git commit -m "プレフィックス: 簡潔な要約" -m "- 詳細な変更点1" -m "- 詳細な変更点2" && git push
  ```
- 分割すべきと判断したケースでは、それぞれの論理的な単位ごとに `git add` と `git commit` を行い、すべて完了した後に `git push` を実行します。

### 6. 結果の報告
- プッシュが成功したら、実際に使用したコミットメッセージ（複数ある場合はすべて）と、作業が完了した旨をユーザーに日本語で簡潔に報告します。
