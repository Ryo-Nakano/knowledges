# Deep Researcher

ユーザーから指定されたトピックについて、Web検索を活用して多段的に深く調査し、結果をMarkdownドキュメントとしてまとめるスキル。AIが浅い検索で妥協することを防ぎ、徹底的な深掘りと情報の裏付けを行うための厳格なガイドラインに従うこと。

## 基本方針

1.  **事実と一次ソースの絶対視**: 自身の事前知識や検索エンジンの「スニペット（概要）」だけで判断してはならない。必ず `web_fetch` を用いて一次ソース（公式ドキュメント、論文、GitHubリポジトリ等）の内容を読み込み、事実確認を行うこと。
2.  **情報のクロスチェック（交差検証）**: 1つのソースの情報のみを鵜呑みにせず、必ず複数の独立したソースを参照して情報が一致しているか、矛盾がないかを確認すること。
3.  **情報源の明示**: 作成するドキュメントには、必ず参考にしたURLと引用元を明記する。

## ワークフローと厳格な実行ガイドライン

### 1. 調査計画と初期検索（仮説構築）
*   ユーザーの要望から、調査の目的と最終的なレポートに必要な要素（What, Why, How, Pros/Cons など）を分解する。
*   初期キーワードを複数設定し、`google_web_search` を実行する。
*   **【必須】**: 日本語だけでなく、英語のクエリや専門用語を用いたクエリなど、最低2〜3種類の異なるアプローチで広く情報を収集すること。

### 2. 深掘り調査 (Deep Research) - ※最も重要なフェーズ
初期検索の結果を元に、以下の手順を**反復（ループ）**する。

1.  **一次ソースの特定と解析**:
    *   検索結果から、信頼性の高い情報源のURLを**必ず3つ以上**抽出する。
    *   抽出したURLに対し `web_fetch` を実行し、具体的な仕様、コード、背景などを読み込む。（プロンプトには「〜に関する詳細な仕様を抽出して」など具体的に指示すること）
2.  **情報ギャップの特定と追加検索**:
    *   フェッチした内容を分析し、「まだ明確になっていない点」「新たに浮上した専門用語」「ソース間で矛盾している点」を特定する。
    *   情報ギャップを埋めるための新たな検索クエリを生成し、再度 `google_web_search` と `web_fetch` を実行する。
3.  **バージョンの確認**:
    *   技術的なトピックの場合、得られた情報が「いつの時点の」「どのバージョン」のものかを確認し、常に最新の情報を探し求めること。

### 3. 調査の終了判定（Exit Criteria）
以下のすべての条件を満たすまで、手順2の深掘り調査を終了してはならない。
*   [ ] ユーザーの質問に対する直接的な回答が、一次ソースの裏付けとともに得られているか？
*   [ ] その回答の「理由（Why）」や「具体的な実現方法（How）」まで説明できるか？
*   [ ] 複数のソースによるクロスチェックが完了し、最新の状況であることが確認できたか？

### 4. ドキュメントの生成
すべての調査が完了したら、収集・分析した情報を構造化し、以下の要素を含むMarkdown形式のレポートを作成する。
*   **タイトル**
*   **エグゼクティブサマリー (Summary)**: 調査結果の簡潔な結論。
*   **詳細な調査結果 (Findings)**: 深掘りした内容、背景、技術的な詳細など。
*   **未解決の懸念事項 (Limitations/Open Questions)**: 徹底的に調査しても分からなかった点や、今後の動向による不確実性があれば記述する。
*   **情報源 (References)**: 使用したURLのリスト。

*   `write_file` を使用して、指定された場所（指定がない場合は `research/` などのディレクトリ）にファイルを保存する。
