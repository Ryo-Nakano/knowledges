# 日報生成スキル (Daily Report Generator)

このスキルは、プロジェクト内のタスク進行状況を自動的にスキャンし、所定のフォーマットで日報を作成するために使用します。

## トリガー条件
ユーザーから「日報を作成して」「今日の日報を書いて」といった依頼があった場合。

## 処理手順

1. **情報収集**:
   - `tasks/` および `tasks/done/` ディレクトリ配下にある直近のタスクフォルダを検索します（主に本日の日付が付いたタスクディレクトリ）。
   - 各ディレクトリ内の `TODO.md`、`MEMO.md` などのタスク管理ファイルを読み込みます（必要に応じて `SPEC.md` なども確認）。
   - 完了したタスクや進捗があった項目を「今日やったこと」として簡潔に箇条書きで抽出します。
   - 未完了のタスクや次に取り組むべきタスクを「明日やること」として抽出します。

2. **日報の組み立て**:
   - 抽出した情報をもとに、以下のMarkdownフォーマットで日報テキストを作成します。
   - **各項目の先頭には、必ず `[プロジェクト名]` または `[タスク略称]` のようなタグラインを付与してください。**
   - ※「一言」の項目はユーザーが自身で記入するため、何も書き込まずに空のリスト（`- `）のままにしておいてください。

   ```markdown
   今日やったこと
   - [プロジェクト名] 作業内容の要約
   - [プロジェクト名] 作業内容の要約

   明日やること
   - [プロジェクト名] 作業予定の要約
   - [プロジェクト名] 作業予定の要約

   一言
   - 
   ```

3. **保存先の確認と作成**:
   - プロジェクトルート直下に `daily-reports/` ディレクトリが存在するか確認し、存在しない場合は作成します。
   - ファイル名を `YYYYMMDD.md` (例: 本日が2026年3月5日の場合は `20260305.md`) とし、ファイルへの書き込みを行います。

4. **完了の報告**:
   - ファイルの保存が完了したら、ユーザーに日報を作成したこと（ファイルパス）を報告し、「一言」の項目への追記を促してください。

## 注意事項
- 情報元にない作業内容を推測で捏造しないでください。ファイルから確認できる事実に基づいて記載してください。
- ユーザーに都度確認を求めることなく、ファイルの生成・保存まで一気通貫で行ってください。
